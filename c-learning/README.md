# C 语言入门实战 —— 7 个项目从零学会 C 语言

## 👋 欢迎

这个仓库包含 7 个循序渐进的 C 语言实战项目，专为零基础初学者设计。每个项目都是一个可以独立运行的控制台程序，代码中有详细的中文注释解释每个知识点。

**学习理念**：不背语法，动手写代码。通过构建真实的、有趣的小程序来掌握 C 语言。

## 🛠️ 环境准备

### Windows 用户

1. **安装 GCC 编译器**（如果还没有）：

   ```cmd
   winget install BrechtSanders.WinLibs.POSIX.UCRT
   ```

   安装完成后**重新打开终端**，然后验证：

   ```cmd
   gcc --version
   ```

   如果提示找不到 `gcc`，请关闭终端后重新打开，或参考 FAQ。

2. **下载本项目**（如果还没有）：

   把整个 `c-learning/` 文件夹放在你电脑上的任意位置。

## ▶️ 如何运行

### 方法一：命令行编译（推荐用来学习）

打开终端（命令提示符或 PowerShell），进入项目目录：

```cmd
cd 01-hello-calc
gcc main.c -o main.exe
main.exe
```

### 方法二：使用 build.bat

每个项目文件夹里都有一个 `build.bat`，双击即可自动编译并运行。

### 方法三：从根目录批量编译

在 `c-learning/` 根目录下，运行 `build.bat` 可以一次性编译所有 7 个项目。

## 📖 学习建议

### 如果你是纯新手

建议按顺序学，每天完成 1 个项目（大约 30-60 分钟）：

1. **先看注释**：每个 `main.c` 开头都有该项目的学习目标
2. **读一遍代码**：代码中有大量中文注释，先理解每一行在做什么
3. **编译运行**：看看程序跑起来是什么样子
4. **动手改**：尝试修改代码，比如把 `*` 改成 `+`，看会发生什么
5. **做练习**：完成每个项目末尾的思考题

### 如果你有其他语言基础

可以从项目 4 或 5 开始，重点关注 C 语言特有的概念：指针、手动内存管理、字符数组。

## ❓ 常见问题

### Q: 运行 `gcc` 提示"不是内部或外部命令"？

**A:** GCC 没有正确添加到系统 PATH。
- 方法1：重新打开终端（安装后需要刷新环境变量）
- 方法2：手动把 `C:\Users\<你的用户名>\AppData\Local\Microsoft\WinGet\Packages\BrechtSanders.WinLibs.POSIX.UCRT_*\mingw64\bin` 添加到系统 PATH

### Q: 编译时报错 `undefined reference to ...`？

**A:** 检查代码是否正确包含了头文件（`#include <stdio.h>` 等）。如果是在项目 2 中，确保加了 `#include <stdlib.h>` 和 `#include <time.h>`。

### Q: `scanf` 输入时程序行为异常？

**A:** `scanf` 会在缓冲区残留换行符。在读取字符（`%c`）或字符串前，可以多加一个 `getchar()` 来消耗它。

### Q: 中文注释乱码？

**A:** 确保用 UTF-8 编码保存文件。如果用记事本打开，选择"另存为"→ 编码选"UTF-8"。
